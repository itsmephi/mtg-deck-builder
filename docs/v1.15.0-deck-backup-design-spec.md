## Theme

Insurance before infrastructure. All deck data lives in `localStorage` — one cache clear and everything is gone. This feature gives users a one-click full-fidelity backup and restore, doubling as a machine-transfer mechanism (laptop → iPad, etc.). Lives in Settings Hub because it's an infrequent safety-net action, not a daily workflow tool. Becomes partially redundant once cloud save ships, but manual local backup may persist as a user preference.

## 1. Backup File Format

### Structure

```json
{
  "app": "thebrewlab",
  "version": "1.15.0",
  "exportedAt": "2026-03-27T15:42:00.000Z",
  "decks": [ ...Deck[] exactly as stored in localStorage under mtg_builder_decks... ]
}
```

- `app` — fixed string `"thebrewlab"`. Used for validation on restore.
- `version` — the `APP_VERSION` at time of export. Informational for future migration if the data model changes.
- `exportedAt` — ISO 8601 timestamp. Displayed in the restore confirmation modal.
- `decks` — the raw `Deck[]` array. Full fidelity — every `DeckCard` with all Scryfall metadata, quantities, owned counts, sideboards, format, commanderId.

### Filename

`thebrewlab-backup-YYYY-MM-DD.json`

Date is local date at time of export. Spaces replaced with hyphens. Example: `thebrewlab-backup-2026-03-27.json`.

### Scope

All decks in a single file. This is not a per-deck export — the existing text-format export in the sidebar handles that use case.

## 3. Backup Flow

### Trigger

User clicks "Backup All Decks" button.

### Steps

1. Read `decks` from `useDeckManager` context (already in memory — no need to re-read localStorage).
2. Build the backup object: `{ app: "thebrewlab", version: APP_VERSION, exportedAt: new Date().toISOString(), decks }`.
3. Serialize to JSON: `JSON.stringify(backup, null, 2)` (pretty-printed for human readability if someone opens the file).
4. Create Blob: `new Blob([json], { type: "application/json" })`.
5. Trigger download via the same `URL.createObjectURL` + click pattern used by `exportDeck`.
6. Store timestamp: `localStorage.setItem("mtg-last-backup", new Date().toISOString())`.
7. Show app-wide toast: `"Backed up X decks"` where X is `decks.length`.

### No confirmation needed — backup is non-destructive.

---

## 4. Restore Flow

### Trigger

User clicks "Restore from Backup" button.

### Steps

1. Open native file picker via hidden `<input type="file" accept=".json">`.
2. If user cancels the file picker → nothing happens.
3. Read selected file as text via `FileReader`.
4. Parse JSON. If parsing fails → show inline error (Section 5).
5. Validate:
   - `parsed.app === "thebrewlab"` — if not, inline error.
   - `Array.isArray(parsed.decks)` — if not, inline error.
   - `parsed.decks.length > 0` — if empty, specific inline error.
6. If valid → open Restore Confirmation Modal (Section 6).

## 5. Inline Error States

Errors display as a `text-xs text-red-400 mt-2` paragraph below the button row, above the "Last backup" timestamp. Error clears when the user clicks either button again.

| Condition | Message |
|---|---|
| JSON parse failure | "Couldn't read this file. Make sure it's a valid backup." |
| Missing `app: "thebrewlab"` | "This doesn't look like a TheBrewLab backup file." |
| `decks` not an array | "This doesn't look like a TheBrewLab backup file." |
| `decks` is empty array | "This backup contains no decks." |

## 7. Restore Execution

After user confirms in the modal:

1. Parse the backup `decks` array. Apply the same migration logic that `useDeckManager` uses on load — ensure every deck has `format` (default `"freeform"`), every card has `ownedQty`, etc. This handles restoring backups made from older app versions.
2. Replace the decks array in `useDeckManager` state. This needs a new method on the context: `replaceAllDecks(newDecks: Deck[])`.
3. Set active deck to the first deck in the restored set: `setActiveDeckId(newDecks[0].id)`.
4. Close the restore modal.
5. Close the Settings Hub (`onClose()` or equivalent).
6. Show app-wide toast: `"Restored {count} decks"`.

### `replaceAllDecks` method

New method on `DeckContextType`:

```ts
replaceAllDecks: (newDecks: Deck[]) => void;
```

Implementation in `DeckProvider`:

```ts
const replaceAllDecks = (newDecks: Deck[]) => {
  setDecks(newDecks);
  if (newDecks.length > 0) {
    setActiveDeckIdState(newDecks[0].id);
  } else {
    setActiveDeckIdState(null);
  }
  setDeckViewMode("main");
};
```

This triggers the existing `useEffect` that writes `decks` to localStorage, so persistence is automatic.

## 8. App-Wide Toast System

### Current State

SearchWorkspace has a local toast: `toastMessage` state, `showToast` callback, `toastTimerRef`, and a `fixed bottom-4 right-4 z-50` div. This gets lifted to `page.tsx`.

### New Architecture

**State in `page.tsx`:**

```ts
const [toastMessage, setToastMessage] = useState<string | null>(null);
const toastTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

const showToast = useCallback((message: string) => {
  if (toastTimerRef.current) clearTimeout(toastTimerRef.current);
  setToastMessage(message);
  toastTimerRef.current = setTimeout(() => setToastMessage(null), 2000);
}, []);
```

**Toast render in `page.tsx`** (at the bottom of the return, outside all conditional views):

```tsx
{toastMessage && (
  <div className="fixed bottom-4 right-4 z-50 px-3 py-2 bg-surface-raised border border-line-default rounded-lg text-xs text-content-heading shadow-lg max-w-xs">
    {toastMessage}
  </div>
)}
```

### Props Threading

- `showToast` passed to `SettingsView` (for backup/restore feedback)
- `showToast` passed to `SearchWorkspace` (replaces its local toast)
- Available for any future component that needs transient feedback

### SearchWorkspace Migration

Remove from SearchWorkspace:
- `toastMessage` state
- `toastTimerRef` ref
- `showToast` callback
- The toast render div at the bottom of the component

Add to SearchWorkspace props:
- `showToast: (message: string) => void`

All existing `showToast(...)` calls inside SearchWorkspace remain unchanged — they now call the prop instead of the local function.

## 10. useDeckManager Changes

### New Export

Add `replaceAllDecks` to `DeckContextType` interface and implement in `DeckProvider` (see Section 7).

### Migration Utility

Extract the existing deck migration logic (adding `format`, `ownedQty` defaults) into a reusable function so both the initial load and `replaceAllDecks` can call it:

```ts
function migrateDecks(rawDecks: any[]): Deck[] {
  return rawDecks.map((deck: any) => ({
    ...deck,
    format: deck.format ?? "freeform",
    commanderId: deck.commanderId ?? undefined,
    cards: deck.cards.map((card: any) => {
      if (card.ownedQty !== undefined) return card;
      return { ...card, ownedQty: card.isOwned ? card.quantity : 0 };
    }),
  }));
}
```

The existing `useEffect` load logic calls `migrateDecks(parsedDecks)`. The new `replaceAllDecks` also calls `migrateDecks(newDecks)` before setting state.
