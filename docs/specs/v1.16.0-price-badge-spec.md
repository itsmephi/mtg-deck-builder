# Price Badge on Card Tiles — Design Spec

**Backlog item:** "Card price display missing from persistent grid tile — deferred from v1.3.0 design"
**Prototype:** `docs/v1.16.0-price-badge-prototype.html`
**Status:** Approved

---

## Overview

Two changes to `VisualCard.tsx`:

1. **Price pill badge** — always-visible price on every grid tile, both modes
2. **Unified hover overlay** — deck overlay gains name + type (matching search); search overlay loses its price line (now redundant)

---

## 1. Price Pill Badge

### Position & visual

Straddling bottom-right corner of the card tile, same visual language as the qty pill (which straddles bottom-center). Sits *outside* the `overflow: hidden` inner container, same as qty pill.

- `absolute`, `bottom: -7px`, `right: -4px`, `z-22`
- `bg-black/82`, `backdrop-blur-[8px]`, `text-[11px] font-semibold tabular-nums`
- `rounded-[7px]`, `px-[7px] py-[3px]`
- `shadow-md` (same as qty pill: `0 2px 6px rgba(0,0,0,0.5)`)
- `border: 1px solid rgba(255,255,255,0.08)` — subtle edge definition
- `whitespace-nowrap`, `leading-tight`

### Content

- Has price: `$X.XX` (from `card.prices?.usd`)
- No price: `—` (em dash) at reduced opacity (`opacity-45`)

### Scaling by tile size

The badge should scale its font size per tile size. The tile size isn't directly available as a prop today — Claude Code should determine the best way to pass or derive it. Target sizes:

| Tile size | Font | Padding | Bottom | Right |
|-----------|------|---------|--------|-------|
| XS | 9px | 2px 4px | -6px | -3px |
| S | 10px | 2px 6px | -6px | -3px |
| M | 11px | 3px 7px | -7px | -4px |
| L | 12px | 3px 8px | -7px | -4px |
| XL | 13px | 4px 9px | -7px | -4px |

### Hover behavior

Fades out when the slide-up overlay is active — same pattern as qty pill: `opacity-0 pointer-events-none` with `transition-opacity duration-150` on group-hover.

### Badge coexistence

No conflicts with existing badges:
- Crown badge: top-left — no overlap
- Warning badge: top-right — no overlap
- × remove: top-right inset, hover-only — no overlap
- Qty pill: bottom-center — clear separation even at XS
- In-deck dot (search): top-left — no overlap

### Both modes

Identical rendering in `mode="deck"` and `mode="search"`. In search mode there's no qty pill, so the price pill sits alone at the corner.

---

## 2. Unified Hover Overlay

### Deck mode overlay — add name + type

Insert card name and type line *above* the existing warning bar / qty controls. New content order:

1. **Card name** — `text-xs font-semibold text-content-primary text-center leading-tight line-clamp-2`
2. **Type line** — `text-[10px] text-content-tertiary truncate w-full text-center`
3. Warning bar (existing, unchanged)
4. Qty controls (existing, unchanged)
5. Owned counter (existing, unchanged)

Style the name and type identically to how they appear in the search overlay today.

### Search mode overlay — remove price line

Remove the `<span>` that shows price between the type line and the "+ Add to Deck" button. The persistent price pill makes it redundant.

Updated search overlay order:
1. Card name (existing)
2. Type line (existing)
3. "+ Add to Deck" button (existing)

---

## Out of Scope

- List view (already shows price in a column)
- CardModal changes
- Price formatting changes (keep existing `$X.XX` format)
- Any changes to the qty pill itself
