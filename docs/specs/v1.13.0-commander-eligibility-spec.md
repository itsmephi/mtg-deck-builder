# v1.13.0 Design Spec — Commander Eligibility Fixes + Vehicle/Spacecraft Support

## Overview

This release addresses two bugs with reversible card printings (Secret Lair display commanders) and adds support for the July 2025 Commander rules change allowing legendary Vehicles and Spacecraft as commanders.

---

## Scope

| Item | Type | Description |
|------|------|-------------|
| Reversible card commander eligibility | Bug | Secret Lair reversible printings (e.g., Jinnie Fay SLD #1510) fail the `isEligibleCommander` check — crown shows dim with "Must be Legendary" tooltip despite being legendary |
| Variant picker empty after swap | Bug | After swapping to a reversible printing, the Swap Art tab shows no variants because `oracle_id` lookup fails for reversible cards |
| Vehicle/Spacecraft commander support | Enhancement | Legendary Vehicles and Spacecraft with printed P/T are now eligible commanders per July 2025 rules update |
| ScryfallCard type extension | Chore | Add `power` and `toughness` fields to support P/T check |

## Bug 1: Reversible Card Commander Eligibility

### Root Cause

Reversible cards (layout `"reversible_card"`) store `type_line` differently than standard cards. For these printings:
- Root-level `type_line` may be a combined format: `"Legendary Creature — Elf Druid // Legendary Creature — Elf Druid"`
- Or root-level `type_line` may be `undefined`, with the actual type lines in `card_faces[0].type_line` and `card_faces[1].type_line`

The current `isEligibleCommander` only checks root `type_line`, missing the `card_faces` case.

### Fix

Update `isEligibleCommander` to check:
1. Root `type_line` (handles combined format and standard cards)
2. `card_faces[0].type_line` as fallback (handles reversible cards where root is undefined)

Same logic applies to `oracle_text` — check root first, then `card_faces[0].oracle_text`.

## Bug 2: Variant Picker Empty After Swap

### Root Cause

`getCardPrintings(oracleId)` searches by `oracle_id`. Reversible cards may have a different `oracle_id` than standard printings of the same card, or the field may be malformed. This causes the Scryfall search to return no results.

### Fix

Change `getCardPrintings` to search by exact card name instead of `oracle_id`:
- Use Scryfall query: `!"${cardName}" unique:prints` (exact name match across all printings)
- This reliably returns all printings regardless of layout type

### API Change

```typescript
// Before
export async function getCardPrintings(oracleId: string): Promise<ScryfallCard[]>

// After
export async function getCardPrintings(cardName: string): Promise<ScryfallCard[]>
```

Update CardModal to pass `previewCard.name` instead of `previewCard.oracle_id`.

**Edge case:** Double-faced cards have names like `"Delver of Secrets // Insectile Aberration"`. The exact name search handles this correctly — Scryfall matches the full combined name.

## Enhancement: Vehicle/Spacecraft Commander Support

### Rules Background

As of July 25, 2025 (Edge of Eternities release), the Commander rules were updated:
- Legendary Vehicles with printed power/toughness can be your commander
- Legendary Spacecraft with printed power/toughness can be your commander
- Source: [WotC Edge of Eternities Mechanics Article](https://magic.wizards.com/en/news/feature/edge-of-eternities-mechanics)

### Implementation

Update `isEligibleCommander` in `formatRules.ts`:

```typescript
export function isEligibleCommander(card: DeckCard): boolean {
  // Get type_line and oracle_text, checking card_faces as fallback for reversible cards
  const typeLine = card.type_line ?? card.card_faces?.[0]?.type_line ?? '';
  const oracleText = card.oracle_text ?? card.card_faces?.[0]?.oracle_text ?? '';
  
  const isLegendary = typeLine.includes("Legendary");
  
  // Traditional: Legendary Creature
  const isLegendaryCreature = isLegendary && typeLine.includes("Creature");
  
  // Explicit commander text (e.g., Grist, the Hunger Tide)
  const hasCommanderText = oracleText.includes("can be your commander");
  
  // July 2025 rule: Legendary Vehicle or Spacecraft with P/T
  const isLegendaryVehicle = isLegendary && typeLine.includes("Vehicle");
  const isLegendarySpacecraft = isLegendary && typeLine.includes("Spacecraft");
  const hasPowerToughness = card.power !== undefined && card.toughness !== undefined;
  const isEligibleArtifact = (isLegendaryVehicle || isLegendarySpacecraft) && hasPowerToughness;
  
  return isLegendaryCreature || hasCommanderText || isEligibleArtifact;
}
```

### New Helper Function

Add a helper to identify Vehicle/Spacecraft for tooltip purposes:

```typescript
export function isVehicleOrSpacecraftCommander(card: DeckCard): boolean {
  const typeLine = card.type_line ?? card.card_faces?.[0]?.type_line ?? '';
  const isLegendary = typeLine.includes("Legendary");
  const isVehicle = typeLine.includes("Vehicle");
  const isSpacecraft = typeLine.includes("Spacecraft");
  const hasPT = card.power !== undefined && card.toughness !== undefined;
  return isLegendary && (isVehicle || isSpacecraft) && hasPT;
}
```

## UI: Info Icon for Vehicle/Spacecraft

### Behavior

When hovering the crown badge on an eligible Vehicle or Spacecraft:
- Tooltip text: `"Set as Commander"` followed by a small info icon
- Info icon is a clickable `ⓘ` that opens the WotC mechanics article in a new tab
- Link: `https://magic.wizards.com/en/news/feature/edge-of-eternities-mechanics`

### Grid View (VisualCard.tsx)

Crown badge tooltip for Vehicle/Spacecraft:

```
Set as Commander ⓘ
```

The `ⓘ` is a clickable link. Clicking the crown itself sets commander. Clicking `ⓘ` opens the link.

**Implementation approach:** The tooltip becomes interactive. Use a small popover instead of a native `title` attribute, containing the text and a linked info icon.

### List View (ListCardTable.tsx)

Same behavior — crown icon tooltip shows `"Set as Commander ⓘ"` with clickable info link for Vehicle/Spacecraft cards.

### Visual Spec

| Element | Style |
|---------|-------|
| Info icon | `ⓘ` character or `Info` lucide icon at 10px |
| Icon color | `text-blue-400` |
| Hover | `text-blue-300 underline` |
| Cursor | `pointer` on the icon |
| Link target | `_blank` with `rel="noopener noreferrer"` |

## Type Extension: ScryfallCard

Add `power` and `toughness` to `ScryfallCard` in `types/index.ts`:

```typescript
export interface ScryfallCard {
  // ... existing fields ...
  power?: string;      // e.g., "3", "*", "1+*"
  toughness?: string;  // e.g., "3", "*", "1+*"
}
```

Note: Scryfall returns these as strings (not numbers) to handle variable P/T like `*` or `1+*`.
