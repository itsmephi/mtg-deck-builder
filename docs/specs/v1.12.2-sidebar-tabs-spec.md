# v1.12.2 ŌĆö Sidebar Tabs: Seamless Tab Treatment

> **Status:** DESIGN APPROVED
> **Date:** 2026-03-23
> **Issue:** #75 ŌĆö Sidebar tabs: raised appearance and missing blue active indicator bar
> **Designed in:** Claude Chat design session + interactive prototypes

---

## Summary

Replace the current sidebar tab styling (blue underline indicator on active tab, no background differentiation) with a seamless physical tab metaphor. The active tab blends into the sidebar body; the inactive tab recesses behind it. No blue indicator bar.

---

## Design

### Active Tab

- **Background:** `bg-surface-panel` ŌĆö matches the sidebar body, creating a seamless connection
- **Text color:** `text-content-primary`
- **Bottom border:** `border-b border-transparent` ŌĆö no visible edge, tab flows into content
- **Icon color:** inherits `text-content-primary`

### Inactive Tab

- **Background:** `bg-surface-deep` ŌĆö darker than the sidebar surface, looks recessed/behind
- **Text color:** `text-content-muted`
- **Bottom border:** `border-b border-line-subtle` ŌĆö creates the shelf edge where the active tab sits on top
- **Icon color:** inherits `text-content-muted`

### Inactive Tab ŌĆö Hover

- **Background:** `bg-surface-panel` ŌĆö lifts to match sidebar surface (previews the active state)
- **Text color:** `text-content-secondary` ŌĆö brighter than muted, but not as strong as primary
- **Bottom border:** remains `border-b border-line-subtle` (doesn't disappear on hover)
- **Transition:** `transition-colors` (same as current)

### Collapse Button (Desktop)

- Treated as inactive/recessed: `bg-surface-deep`, `text-content-muted`, `border-b border-line-subtle`
- Hover: `text-content-primary` (text brightens, no bg change ŌĆö it's a utility button, not a tab)

### Settings Gear (Mobile)

- Same treatment as collapse button: `bg-surface-deep`, `text-content-muted`, `border-b border-line-subtle`
- Hover: `text-content-primary`

### Container Changes

- **Remove** the `border-b border-line-subtle` from the tab bar container div
- The bottom edge is now created by the individual inactive tabs' and utility buttons' `border-b` ŌĆö the active tab's transparent border breaks the line, creating the seamless illusion

## Token Mapping ŌĆö Both Themes

| Element | Token | Warm Stone | Zed Dark |
|---|---|---|---|
| Active tab bg | `surface-panel` | `#231f1d` | `#21252b` |
| Inactive tab bg | `surface-deep` | `#15120f` | `#1b1f24` |
| Active text | `content-primary` | `#e7e0d6` | `#d7dae0` |
| Inactive text | `content-muted` | `#6b6259` | `#636d83` |
| Hover text | `content-secondary` | `#a69e94` | `#9da5b4` |
| Inactive border-b | `line-subtle` | `#292524` | `#21252b` |

No new tokens required. `surface-deep` is always darker than `surface-panel` in both themes, so the recessed effect works universally.
