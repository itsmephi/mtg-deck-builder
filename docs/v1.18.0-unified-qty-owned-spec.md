# v1.18.0 — Unified Qty/Owned Input

**Milestone:** v1.18.0 — Card Management
**Prototype:** `docs/v1.18.0-unified-qty-owned-prototype.html` (v4)
**Scope:** VisualCard overlay redesign, ListCardTable column restructure, badge behavior changes, price badge conditional color, new data field

---

## Overview

Replace the separate qty stepper and owned counter with a unified `owned / qty` control in both grid and list views. Add a checkmark toggle for quick "own all" with contextual logic. Make the grid qty badge transform into a toggle on hover. Conditionally color the price badge based on ownership.

---

## Data Model

### New field: `isOwned` boolean on `DeckCard`

Add `isOwned: boolean` (default `false`) alongside existing `ownedQty: number`.

- `isOwned` = the checkbox/badge toggle state (is the user tracking ownership?)
- `ownedQty` = the actual number owned (fine-tuned via steppers or inline edit)

**Toggle logic (same for grid badge and list checkbox):**
- First activation (`isOwned` was false AND `ownedQty === 0`): set `ownedQty = quantity`, set `isOwned = true`
- Deactivation (`isOwned` was true): set `isOwned = false`, **retain** `ownedQty` value
- Re-activation (`isOwned` was false AND `ownedQty > 0`): set `isOwned = true`, `ownedQty` unchanged

**Derived states:**
- Fully owned: `isOwned === true && ownedQty >= quantity`
- Partially owned: `isOwned === true && ownedQty > 0 && ownedQty < quantity`
- Not owned: `isOwned === false` (regardless of ownedQty value)

**Tooltips (both grid badge-as-checkmark and list checkbox):**
- When `isOwned === false`: "Mark as owned"
- When `isOwned === true`: "Unmark as owned"

### Migration

- Existing decks have `ownedQty` but no `isOwned` field
- Migration in `migrateDecks()`: if `ownedQty > 0`, set `isOwned = true`; otherwise `isOwned = false`

## Grid View — VisualCard

### Qty badge at rest (no hover)

Position: center-bottom straddling the card edge (unchanged from current). Shows `quantity` number.

**Badge appearance by ownership state:**

| State | Background | Text | Border |
|-------|-----------|------|--------|
| Not owned | `rgba(30, 28, 24, 0.85)` | `text-primary` | `rgba(255,255,255,0.12)` |
| Partially owned | `rgba(30, 28, 24, 0.85)` | green | `rgba(74,222,128,0.4)` |
| Fully owned | `rgba(74,222,128,0.7)` | `#fff` | `transparent` |

Badge is **not interactive** at rest — no click handler, default cursor. Visual-only indicator.

### Badge transform on hover

On card hover, the badge animates upward to straddle the **top edge of the overlay** and transforms into a circular ✓ toggle button.

**Animation:** `transition: all 0.25s cubic-bezier(0.4, 0, 0.2, 1)`. Badge moves from bottom-center to overlay-top-center, border-radius becomes circle, content swaps from quantity number to ✓ checkmark.

**Content swap:** `.badge-number` visible at rest, `.badge-check` (✓) visible on card hover. CSS toggle via `.card-tile:hover`.

**Badge-as-checkmark hover states:**

| Badge state | Appearance | On badge hover |
|-------------|-----------|----------------|
| Not owned | Dark bg, muted border, muted ✓ | Green border, green dim bg, green ✓ |
| Partially owned | Green dim bg, green border, green ✓ | Red border, red dim bg, red ✓ |
| Fully owned | Solid green bg, white ✓ | Red dim bg, red border, red ✓ |

The red hover on partially/fully owned signals "this will unmark" — consistent with destructive-intent pattern used elsewhere in the app.

### Overlay content (replaces current layout)

Current overlay has: card name, type line, `− qty +` stepper, `− owned/qty +` counter (4 elements).

New overlay has: card name, type line, unified number row (3 elements — simpler).

**Unified number row layout:**

```
  −○ [owned] +○  /  −○ [qty] +○
```

- Two number groups separated by a `/` character
- Left group = owned number, right group = qty number
- Circle −/+ buttons on each group: `w-6 h-6 rounded-full`, border `rgba(255,255,255,0.12)`

**Progressive disclosure:** The −/+ circle buttons are hidden by default (`opacity: 0, transform: scale(0.8)`). They fade in (`opacity: 1, transform: scale(1)`) when the cursor hovers the number group area. This keeps the initial overlay read clean — just the numbers and slash.

**Number styling:**
- Owned number: `text-secondary` when partial, `green` when `ownedQty >= quantity`, `text-primary` when 0
- Qty number: always `text-primary` (white)
- Both numbers are inline-editable: click to select, type a value, Enter/blur commits, Escape reverts (same pattern as current qty inline edit)

**Overlay height:** ~40% of card height (unchanged).

### Price badge

- Z-index stays **above** the overlay (z-index: 50) so it remains visible when overlay slides up
- Color: **muted** (`text-secondary`) when not fully owned; **green** when fully owned (`isOwned && ownedQty >= quantity`)
- Position: bottom-right straddling (unchanged)

### Card image dimming

Unchanged behavior — scales with ownership. Full color when not owned, partial dim when partially owned, full dim when fully owned.

### × remove button

Unchanged — top-right, appears on hover, z-index above overlay.

## List View — ListCardTable

### Column restructure

**Current columns:** `[−qty+] [Owned X/Y] [Name] [Type] [Mana] [Price] [×]`

**New columns:** `[Owned] [Qty] [Name] [Type] [Mana] [Price] [×]`

### Owned column

Contains only the checkmark circle toggle button.

- Circle button: same style as v2 prototype — `w-6 h-6 rounded-full`, border `rgba(255,255,255,0.12)`
- Unchecked: transparent bg, muted border, muted ✓ (or invisible ✓ that appears on row hover)
- Checked: green dim bg, green border, green ✓
- Checked + hover: fades toward muted (signals "will unmark")
- Tooltip: "Mark as owned" / "Unmark as owned" based on state
- Same toggle logic as grid badge (first activation sets ownedQty = quantity, subsequent toggles retain value)
- Column header: **Owned**

### Qty column

Contains the `X / Y` readout with hover-revealed −/+ buttons.

- Layout: `−○ [X] +○ / −○ [Y] +○` — same as grid overlay number row but horizontal in a table cell
- −/+ circle buttons appear on **row hover** (same pattern as current qty −/+ buttons)
- X = owned number (left of slash), Y = qty number (right of slash)
- X styling: `text-secondary` when partial, `green` when fully owned
- Y styling: always `text-primary`
- Both X and Y are inline-editable (click to type)
- When `isOwned === false`: owned −/+ buttons and inline edit on X are **disabled** (grayed out, no pointer events). Y −/+ and inline edit remain active.
- When `isOwned === true`: all controls active
- Column header: **Qty**

### Price column

- Color: muted when not fully owned, green when fully owned (matches grid)

### × remove column

Unchanged — appears on row hover.

## Interaction Summary

### Three speeds of owned editing

1. **Quick toggle:** Click the badge-checkmark (grid) or checkbox (list) — one tap to own all or unmark
2. **Fine-tune:** −/+ steppers on the owned number for incremental adjustment
3. **Inline edit:** Click the owned number to type any value — handles high quantities (25 Mountains)

### Locking behavior

When `isOwned === false` (unchecked/unmarked):
- **Grid:** Owned number in overlay is still visible but the −/+ buttons for owned are disabled
- **List:** Owned −/+ disabled, owned inline edit disabled, number grayed

When `isOwned === true` (checked/marked):
- All owned editing controls are active

Qty controls are **always active** regardless of owned state.
