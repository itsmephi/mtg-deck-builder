# v1.19.0 ÔÇö Search List View Toggle

## Overview

Activates the disabled list view button in the SearchWorkspace toolbar. Search results render as a browse-only table ÔÇö no quantity controls, no owned tracking. Reuses existing visual patterns from deck `ListCardTable` (color-tinted rows, mana symbols, cursor-follow thumbnail) and search grid (in-deck green dot, add-to-deck flow).

## Data Model

N/A ÔÇö no new fields or types.

### New localStorage key

| Key | Value | Default |
|-----|-------|---------|
| `mtg-search-view-mode` | `"grid" \| "list"` | `"grid"` |

Persisted on every toggle. Read on mount to restore last-used view.

## Behavior

### View toggle

The existing disabled list button in SearchWorkspace toolbar row 2 becomes functional. Active/inactive button styles match the existing grid button pattern already in the toolbar. View mode persists to localStorage.

### Tile size slider in list mode

When list view is active, `TileSizeSlider` renders as disabled: `opacity-30 pointer-events-none`. Remains in DOM (no layout shift). Re-enables when switching back to grid.

### Sort controls

Unchanged. Sort changes the Scryfall API query ÔÇö view mode only changes rendering.

### Search takeover

Renders regardless of view mode. After dismissal, results render in whichever view mode is active.

## List View Table ÔÇö `SearchListTable`

New component. Renders in the same scroll container position where the card grid currently renders.

### Container

Same wrapper pattern as deck `ListCardTable`: `bg-surface-base border border-line-subtle rounded-lg shadow-sm`. Margin `10px` around the table. `table-fixed` layout.

### Column order and widths

| Column | Width | Content |
|--------|-------|---------|
| + (add) | `w-10` | Hover-reveal circular + button |
| Name | flex fill | In-deck green dot + card name, truncated |
| Type | `w-48` | Type line, truncated |
| Mana | `w-24` | Scryfall SVG symbols (same `renderManaSymbols` as deck `ListCardTable`) |
| Price | `w-20` | Right-aligned `$X.XX` or `"N/A"` |

### Header row

Same style as deck `ListCardTable` header. Column labels: (empty) | Name | Type | Mana | Price.

### Row behavior

**Click row** (except + button): opens `CardModal` with `context='search'`. Same `onNext`/`onPrev` navigation through results array.

**Row background:** Color-tinted by card color ÔÇö same `getRowTint`/`getRowHoverTint` logic and rgba values as deck `ListCardTable`. Color determined from `card.colors` array using the same rules (multi, land, colorless, single color).

**Row height:** Same cell padding as deck `ListCardTable` (`px-2 py-1` with `text-xs` content).

### + add button

- Position: leftmost column, before name
- Circular: `w-[22px] h-[22px] rounded-full border border-white/10`
- Content: `+` character
- Visibility: `opacity-0` default, `opacity-100` on row hover (same pattern as deck list ├Ś button)
- Hover (on button): `bg-blue-500/15 border-blue-500/30 text-blue-400`
- Click: calls existing `handleAdd(card)` with price rescue, shows toast, `event.stopPropagation()`
- Always functional regardless of in-deck status (adds another copy)

### In-deck row appearance

When card name exists in `deckCardNames` Set:
- Green dot before name: `w-1.5 h-1.5 rounded-full bg-green-400` with `box-shadow: 0 0 4px rgba(74, 222, 128, 0.4)`
- Card name: `text-content-tertiary` (dimmed from default `text-content-primary`)
- Type: `text-content-faint` (dimmed from default `text-content-tertiary`)
- Mana symbols: `opacity-40`
- Price: `text-content-faint`
- No checkmark. Green dot is the only in-deck indicator.

### Cursor-follow thumbnail

Controlled by existing `showThumbnail` preference. Same implementation as deck `ListCardTable` hover preview.

### Toolbar controls by view mode

| State | Grid button | List button | Tile slider |
|-------|------------|-------------|-------------|
| Grid active | Active style | Inactive | Enabled |
| List active | Inactive | Active style | Disabled (opacity-30) |

## Intersection States

**In-deck + hover:** Coexist. Text stays dimmed, background shows hover tint, + appears with full interactivity. No precedence conflict.

**SearchTakeover + list mode:** Takeover replaces content area. View mode preserved but irrelevant. After dismissal, renders in persisted mode.

## Edge Cases

- **DFC / split cards:** Mana column uses same DFC handling as deck `ListCardTable` ÔÇö front face cost, `//` separator for split
- **No mana cost (lands):** Empty mana cell
- **Zero results / loading / empty:** Same centered messages as current grid states, rendered regardless of view mode. `SearchListTable` only rendered when `results.length > 0`
- **Null price:** Shows "N/A"
