## Theme

Give the cards room to breathe. Consolidate scattered controls into a collapsible sidebar, slim the workspace toolbar to a single row, and clean up grid view tiles so the card art is the star.

---

## 1. Collapsible Sidebar

### Expanded State

- Width: `w-60` (240px) — narrower than original `w-64` to maximize workspace real estate.
- Contains: tab bar, tab content (Search or Decks), footer.
- Smooth CSS transition on width change; workspace `flex-1` reflows naturally.

### Collapsed State

- Width: 48px icon rail.
- Shows vertically stacked circular icon buttons:
  - 🔍 Search icon — click expands sidebar and activates Search tab.
  - 📑 Layers/Decks icon — click expands sidebar and activates Decks tab.
  - *(flex spacer)*
  - ☕ Coffee link (buy me a coffee).
  - ⚙ Settings icon.
- No text labels in collapsed state; icons only.
- No import/export/shopping icons in collapsed rail — those only exist in the expanded Decks tab.
- A dedicated collapse chevron (`‹`) in the tab bar of the expanded sidebar triggers collapse.

### Behavior

- **Breakpoint:** `md:` and up only. Mobile layout (`< md`) is unchanged — sidebar stacks on top at `h-[40vh]` as today. Collapse functionality is not available on mobile.
- **Persistence:** Collapsed/expanded state saved to `localStorage` under key `mtg-sidebar-collapsed`. Default: expanded (first-time users see the full sidebar).
- **Transition:** `transition: width 300ms cubic-bezier(0.4, 0, 0.2, 1)`. Workspace grid/list reflows smoothly as width animates.
- **Hotkey (stretch):** `Ctrl+B` or `[` to toggle. Nice-to-have, not required for v1.3.0.

## 2. Sidebar Tab System

Two tabs at the top of the expanded sidebar.

### Tab Bar

- Two tabs: **Search** (magnifying glass icon + "Search") and **Decks** (layers icon + "Decks").
- Active tab: blue text + blue bottom border (`text-blue-400 border-blue-500`).
- Inactive tab: `text-neutral-500 border-transparent hover:text-neutral-300`.
- Collapse chevron (`‹`) sits to the right of the tabs in the same row. Clicking it collapses the sidebar and saves state to localStorage.
- Active tab persisted to `localStorage` under key `mtg-sidebar-active-tab` (`"search"` | `"decks"`, default `"search"`).

---

### 2a. Search Tab (default active)

Contains everything the current sidebar search area has, with minor improvements:

**Search field** — existing debounce (500ms), with additions:

- **Clear button (×):** Small circular button, appears inside the search field when text is present. Clicking it clears the field and immediately snaps back to the active category results without re-fetching (use cached results from component state).
- **Placeholder text:** "Search cards..." when idle.

**EDHREC category dropdown** — unchanged functionality for this milestone. Same five categories: Top 50 Trending, Top Creatures, Popular Lands, Top Artifacts, Budget Staples.

**Results list:**

- Each row shows: card name (truncated), **inline mana cost symbols** (small, right-aligned), and circular `+` button on hover.
- No price display in results (deferred).
- Card preview tooltip on hover continues to work as today.

**Result caching:** When user clears search, snap back to last-loaded category results instantly. Cache in component state, not localStorage.

**No actions strip on the Search tab.** Import/export and buy links only appear on the Decks tab.

---

### 2b. Decks Tab

Full-height deck management panel.

**Deck list — each row contains:**

- **Active indicator** — small blue dot (`w-1.5 h-1.5 rounded-full bg-blue-500`) for the active deck. Transparent for inactive decks.
- **Deck name** — text, truncated with ellipsis if too long. Clicking the name calls `setActiveDeckId(deck.id)` and `setDeckViewMode("main")`. Name click always navigates to main view.
- **Card count** — small muted text (`text-[10px] text-neutral-600`), right-aligned.
- **Sideboard layers icon** — circular button, two states:
  - **Muted (no sideboard exists):** `text-neutral-700 hover:text-neutral-400`. Click creates the sideboard for that deck. Icon turns blue. Workspace switches to sideboard view automatically.
  - **Blue/active (sideboard exists):** `text-blue-400`. Click switches workspace to sideboard view for that deck. If the deck is not the active deck, it also becomes the active deck first.
  - The layers icon **never** toggles back to main view. Returning to main view is handled exclusively by the Main/Side pill in the workspace toolbar.
- **× remove icon** — circular button, appears **on hover only** (`opacity-0 group-hover:opacity-100`). Hover turns red (`hover:text-red-400 hover:bg-red-500/10`). Clicking opens a small dropdown with:
  - "Delete Deck" — triggers confirmation dialog.
  - "Delete Sideboard" — only visible if sideboard exists, triggers confirmation dialog.

**Key interaction rule:** Name and layers are independent actions. Clicking a deck name always goes to main. Clicking layers always goes to sideboard (or creates one). No ambiguity.

**"+ New Deck" button:** Fixed at the bottom of the deck list, always visible, does not scroll with the list.

**Rename:** Happens in the workspace toolbar (Section 4), not in the sidebar. The sidebar deck list reflects the current name in real time.

### Actions Strip (Decks tab only)

Positioned between the "+ New Deck" button and the footer. **Only visible when the Decks tab is active** — not shown on the Search tab. Compact two-row layout:

```
   Import  ·  Export
  TCGPlayer · Card Kingdom
```

- **Import / Export:** Behave exactly as today. Import opens file picker, export downloads `.txt` for the active deck.
- **TCGPlayer / Card Kingdom:** Behave exactly as today. Open external buy links for unowned cards in the active deck.
- Small, muted button styling (`text-[10px] text-neutral-500 hover:text-neutral-200`). Dot separator between buttons.

## 4. Workspace Toolbar — Slimmed to One Row

### Removed from toolbar (moved to sidebar)

- Deck dropdown (switching, create, delete, sideboard management) → Decks tab.
- Import/Export buttons → Decks tab actions strip.
- Buy links (TCGPlayer, Card Kingdom) → Decks tab actions strip.

### Remaining — single row layout

**Left side (in this order):**

- **Deck name** — editable inline (click to type, Enter to confirm, Escape to cancel). Full workspace width available, no truncation issues. Styled prominently (`text-lg font-bold text-white`).
- **Card count** — e.g. "58 Cards" with existing color logic (green at 60, red above 60).
- **Total value** — e.g. "Value: $127.43".
- **To Buy** — e.g. "To Buy: $43.20" in green. This is the remaining cost after accounting for owned cards (already computed by `useDeckStats` as `remainingCost`).
- **Sideboard count** — shown when sideboard exists and in sideboard view, e.g. "Side: 12 / 15".

**Right side (in this order):**

- **Opening Hand Simulator button** — "Simulator" with dice icon.
- **Main / Side pill toggle** — unchanged behavior, blue highlight on active side.
- **Sort controls + View controls** — grouped in a single bordered container:
  - Sort By dropdown + direction toggle.
  - Group by Type toggle.
  - Grid / List view toggle.

### Visual treatment

- Single row, `flex items-center justify-between`.
- Same dark background treatment as today's toolbar.
- Compact but not cramped — the removal of the deck dropdown, import/export, and buy links frees significant horizontal space.

## 5. Grid View Hover Overlay

### Default State (no hover)

- **Card art fills the entire tile.** No buttons, no progress bars, no name label.
- **Qty badge** — small **circular** badge in the **top-left corner** (`w-6 h-6 rounded-full`), shows the quantity number (e.g. "3").
  - Color follows existing logic: neutral/gray (`bg-black/60`) for ≤3, green (`bg-green-500/80`) at exactly 4, red (`bg-red-500/80`) at 5+.
  - The red badge at 5+ communicates the 4-copy warning — no separate warning badge needed.
- **Nothing else visible.** The art is the star.

### Hover State

- **× remove button** — appears in the **top-right corner** of the card tile. **Circular** (`w-6 h-6 rounded-full`), semi-transparent background, white ×. Hover turns red. Positioned **above and separate from** the bottom overlay. Not persistent — only visible on hover.
- **Bottom overlay** — slides up from the bottom of the card (`translate-y-full → translate-y-0`). Semi-transparent dark background (`bg-black/75 backdrop-blur-sm`). Covers approximately the bottom third to half of the tile (tunable during implementation).

**Overlay contents (top to bottom):**

- **− qty +** controls — circular buttons (`w-7 h-7 rounded-full`), horizontally centered, qty number between them. Same interaction as today.
- **Owned counter** — displayed as `Owned: X/Y` where X = ownedQty, Y = quantity. X can exceed Y (e.g. "5/3") and that is valid. **Inline editable** — clicking the owned number allows typing a new value (same as current inline editing behavior). No separate +/- buttons for owned.
  - Color: green (`text-green-400`) when X ≥ Y (fully owned), neutral/muted (`text-neutral-400`) when X < Y.
  - `cursor-pointer hover:underline` to hint clickability.
  - No progress bar. The `X/Y` text with color is sufficient in this compact overlay.

### Click Behavior

- **Clicking card art** (area above the overlay, or when not hovering) → opens CardModal as today.
- **Clicking overlay controls** (qty buttons, owned counter) → interacts with those controls, does not open the modal.
- **Clicking × in top-right** → removes card from deck. No confirmation dialog (matches current behavior).

### Card Name

- **Not included in the overlay by default.** With larger tiles from reclaimed sidebar space, the card name printed on the art itself should be more readable.
- **Prototyping checkpoint:** If during testing the art-printed name is still too hard to read at actual tile sizes, add a single-line name label at the very bottom of the tile as a fallback.

### Double-Sided Cards

- No conflict with the hover overlay. Card flip behavior exists only in the CardModal, not on the grid tile.
- Double-sided cards show their front face in the grid, same as today.

### Icon Consistency Rule

All interactive circular icons throughout the app should use `rounded-full`. This applies to: qty badge on card tiles, × remove button on card tiles, × delete on deck rows, + button on search results, sidebar rail icons, qty +/- buttons in the overlay. Maintain this circular treatment consistently.

## 8. Search Functionality — No Changes This Milestone

Search queries, Scryfall API calls, EDHREC categories, and debounce behavior are all unchanged. The broader search refactor (natural language queries like "show me all goblins" or "give me mana draining cards", aggregated data sources) is deferred to a dedicated future milestone.

The only search-related changes are:

- Inline mana cost symbols in search result rows (Section 2a).
- Clear button (×) in the search field (Section 2a).
- Category result caching for instant snap-back on clear (Section 2a).
