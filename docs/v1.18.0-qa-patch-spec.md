# v1.18.0 — QA Patch

**Applies to:** v1.18.0 — Card Management (post-implementation fixes)
**Base spec:** `docs/v1.18.0-unified-qty-owned-spec.md`

---

## Fixes

### 1. Badge straddle position (grid)

**Issue:** Badge slides up past the overlay top edge instead of straddling it.

**Fix:** Adjust the `bottom` calc so the badge center sits exactly on the overlay's top edge — half above, half below. If overlay height is 40% of card height, badge `bottom` should equal `40%` minus half the badge height (14px for a 28px badge). Visually verify the badge straddles the boundary.

### 2. Badge/checkmark background opacity (grid)

**Issue:** Partially owned checkmark and other badge states use transparent backgrounds that are hard to see against dark card art.

**Fix:** All badge states — both at rest and as checkmark — must use solid/opaque backgrounds (alpha ≥ 0.85).

**At-rest badge:**

| State | Background | Text | Border |
|-------|-----------|------|--------|
| Not owned | `rgba(30, 28, 24, 0.95)` | `text-primary` | `rgba(255,255,255,0.15)` |
| Partially owned | `rgba(22, 101, 52, 0.85)` | `#fff` | `rgba(74,222,128,0.5)` |
| Fully owned | `rgba(74, 222, 128, 0.85)` | `#fff` | `transparent` |

**Badge-as-checkmark (hover):**

| Badge state | Appearance | On badge hover |
|-------------|-----------|----------------|
| Not owned | Solid dark bg (`rgba(30,28,24,0.95)`), muted border, muted ✓ | Solid green bg (`rgba(22,101,52,0.9)`), green border, green ✓ |
| Partially owned | Solid green bg (`rgba(22,101,52,0.85)`), green border, green ✓ | Solid red bg (`rgba(153,27,27,0.85)`), red border, red ✓ |
| Fully owned | Solid green bg (`rgba(74,222,128,0.85)`), green border, white ✓ | Solid red bg (`rgba(153,27,27,0.85)`), red border, red ✓ |

### 3. Price badge — always neutral

**Issue:** Price turning green when fully owned distracts from the qty badge which already communicates ownership.

**Fix:** Price badge color is always neutral/muted (`text-secondary`) in both grid and list views. Remove any conditional green logic on price.

### 4. Stepper hover states

**Issue:** The −/+ circle buttons have no visual rollover state.

**Fix:** On hover, steppers should show: brighter border (`rgba(255,255,255,0.25)`), brighter text (`#fff`), subtle background fill (`rgba(255,255,255,0.14)`). Applies to both grid overlay steppers and list view steppers.

### 5. Owned editing always enabled

**Issue:** Graying out owned controls when `isOwned === false` prevents users from editing owned count directly. Better to let users increment owned freely, which auto-activates tracking.

**Fix:** Owned steppers and inline edit are **always enabled** regardless of `isOwned` state. New auto-activation rules:

- If `isOwned === false` and user increments owned (via +/inline edit, ownedQty goes from 0 to 1+): automatically set `isOwned = true`
- If `isOwned === true` and user decrements owned to 0 (via − or inline edit): automatically set `isOwned = false`

The checkbox/badge remains a convenience shortcut for bulk toggle (own all / unmark), not a gate.

### 6. Decrement-to-zero bug

**Issue:** Toggling owned on, then decrementing owned counter to 0 leaves the card in a partial-owned visual state instead of resetting to unowned.

**Fix:** Covered by fix #5 — when `ownedQty` reaches 0 via any path (stepper, inline edit), set `isOwned = false`. Badge and card dimming should return to the "not owned" state.
